# Contributing

Thank you for your interest in contributing to **V4 Fitness Hub & Rehab**.

Contributions are welcome and appreciated. Please follow the guidelines below to help keep the project clean, maintainable, and consistent.

---

## Getting Started

1. Fork the repository.
2. Clone your fork.

```bash
git clone https://github.com/yourusername/V4-Fitness-Hub.git
```

3. Create a new branch.

```bash
git checkout -b feature/your-feature-name
```

4. Make your changes.

5. Commit your changes.

```bash
git commit -m "Add: Short description of the feature"
```

6. Push your branch.

```bash
git push origin feature/your-feature-name
```

7. Open a Pull Request.

---

# Coding Standards

## HTML

- Use semantic HTML5 elements.
- Maintain proper indentation.
- Write accessible markup whenever possible.

## CSS

- Keep styles modular.
- Follow the existing architecture:
  - base.css
  - layout.css
  - components.css
  - responsive.css
  - style.css
- Reuse existing variables and utility classes.

## JavaScript

- Keep files modular.
- Use descriptive variable and function names.
- Avoid duplicate code.
- Maintain the existing project structure.

---

# Pull Request Guidelines

Before submitting a Pull Request, please ensure:

- The website works correctly.
- No JavaScript errors appear in the browser console.
- The layout remains responsive.
- Existing functionality is not broken.
- New code follows the project's coding style.

---

# Reporting Issues

When reporting an issue, include:

- Browser name and version
- Operating system
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

---

# Feature Requests

Feature suggestions are welcome.

Please describe:

- The problem you're trying to solve
- Your proposed solution
- Any additional context
